OPAL config builder

Mirrors configs/defaults.json. Submit validates against OpalConfig().initialize(), then download.

simulation
Wall-clock seconds (virtual) to run. Use -1 to run until the workload finishes.
Random seed for reproducibility — controls workload sampling and any tie-breaking randomness.
Initial worker (vLLM instance) count. The autoscaler can grow this up to router.max_workers.
Write per-stage stats JSON and (optionally) plots to simulation-runs/<timestamp>/.
Print a tqdm-style progress line during the run. Off by default for cleaner logs.
model
Folder name under config_dir containing a config.json. Drives architecture & dimensions.
Directory where model config sub-folders live. Resolved relative to project root.
router
Turn the per-second autoscaler on. When off, worker count is fixed at simulation.num_workers.
If the busiest worker's queue exceeds this many in-flight requests, the autoscaler adds a worker.
Virtual seconds the autoscaler waits between deciding to scale up and the new worker being ready.
Upper bound on total workers the autoscaler will spawn.
Routing policy. RoundRobin: cycle. LeastLoaded: pick min-queue worker. Random: uniform. MaxPrefix: prefer worker with the largest matching KV-cache prefix. Balanced: combination policy.
Virtual seconds between router-side infra-event drains (KV events from workers).
Upper bound on infra-events processed per drain. Caps router CPU per tick.
workload (stages)

Multi-stage workloads run in order. Each stage picks a generator type and supplies its own params.

worker

worker_params

Backpressure cap on the per-worker intake queue. 1 ≈ tight backpressure to the router.
Virtual seconds between worker→router status pushes (gpu_util, queue depth, mem_used).
Virtual seconds the worker batches KV-cache events before reporting them upstream.

hw

GPU label used in logs & reports. Cosmetic — actual perf comes from tflops / mem_bw_TBps.
HBM per GPU in GB. Free memory after model load becomes the KV-cache block pool.
Peak compute throughput. Used by the roofline model for prefill latency estimation.
HBM bandwidth in TB/s. Drives the bandwidth-bound side of the roofline (decode latency).
Tensor-parallel degree per worker. KV cache is sharded across TP ranks; effective HBM = memory_gb × tp.

vllm_params

Max concurrent sequences in a single batch (vLLM's continuous-batching cap).
Per-step token budget across prefill+decode. Also caps a single chunked-prefill chunk.
OPAL-only throttle: max waiting requests holding KV-cache fetch blocks at once. Avoids livelock under tiered KV.
Split long prompts into chunks of max_num_batched_tokens so prefill mixes with decode each step.
KV-cache block granularity in tokens. Smaller = less internal fragmentation, more bookkeeping.

inference_params

Latency engine. roofline: analytical, uses tflops/mem_bw_TBps + (a, b) regression. synthetic: fixed mean_latency_secs per step (debugging).
Synthetic-engine constant per-step latency. Ignored by the roofline engine.
Roofline regression coefficient (compute side). Calibrate with opal/regression-fitting/.
Roofline regression coefficient (bandwidth side). Calibrate with opal/regression-fitting/.
kvc
Comma-separated tier names, in fill order: CPUMemory, LocalNVMe, DistributedFS. Spillover goes to the next tier.
KV-cache chunking granularity for the prefix-token database (tokens per chunk).
When on, partial trailing chunks are persisted (better hit rate, more storage churn).

CPUMemory (fastest tier)

Total bandwidth of the tier; fair-shared across concurrent transfers.
Per-request fixed latency. Requests with size ≤ bw × latency finish in this window without bandwidth contention.
Max parallel in-flight requests (SimPy resource cap). High = effectively unlimited.
Tier size. When full, new entries spill to the next tier in kvc_tiers.

LocalNVMe (mid tier)

Local NVMe read bandwidth. Slower than CPUMemory; faster than DistributedFS.
Per-request fixed latency. NVMe ≈ tens of µs per access.
Max parallel in-flight requests against the device.
Tier size in GB.

DistributedFS (slowest tier, singleton)

Aggregate DFS bandwidth. Shared across all workers in the simulation.
Per-request fixed latency. DFS ≈ hundreds of µs.
Max parallel in-flight requests across the cluster.
Tier size in GB. Default ≈ 1 PB — effectively infinite for most experiments.
